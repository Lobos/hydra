新建Api

Route

后端请求的路由地址，可以通过 QueryString 传入参数，也可以通过 path 传入，下面两条路由对应的 url 获取到的数据是相同的

route: /authorlist/:page/:size
url:   /authorlist/1/10

route: /authorlist
url:   /authorlist?page=1&size=10
Method

API 支持 GET POST PUT DELETE 四种方法。

GET 请求时，Graphql 的入参会从queryString 或者 path 里面获取，其它三个请求时，入参从form body中获取。

权重

当两条路由冲突时，优先使用权重高的路由。

接口类型

如果接口类型选择为 静态数据，接口会原封不动输出 静态数据输入框中填入的数据，如果选择为 Graphql Query，会按 Graphql 查询语句输出数据。
